Gulp - это сборшик задач, инструмент для автоматизации рутинных задач, которые возникают при веб-разработке.
    
Ссылки:
    Сайт gulp: https://gulpjs.com/
    https://www.npmjs.com/ - модули для gulp.

Для работы с Gulp требуется установленная платформа для работы на языке JS - Node.js. Устанавливаем более позднюю версию, т.к. она более стабильная.
    Ссылка - https://nodejs.org/ru
После требуется установка пакета Gulp на компьютер через терминал.
    npm install --global gulp-cli (npm i -g gulp-cli) 
        - где: 
        npm - это платформа; 
        install - команда установки; 
        global - глобально на пк, такая установка использоуется на устройство только один раз; 
        gulp-cli - сам пакет Gulp.

Package.json - файл с конфигами. Содержит в себе настройки текущей сборки, подключеные файлы и плагины.

Плагины Gulp:
    browser-sync - автоматическая перезагрузка страницы, после изменений. Работает как личный сервер.
    gulp-autoprefixer - браузерные префиксы для управления браузерными префиксами в проекте.
    gulp-cssbeautify - преобразует css-файл в удобный и читабельный. (не обязательный).
    strip-css-comments - удаляет комментарии в css. (не обязательный).
    gulp-rename - переименование файлов.
    sass - модуль sass, нужен для работы gulp-sass.
    gulp-sass - компилятор sass в css.
    gulp-cssnano - Минимизирует CSS.
    gulp-uglify - Минимизирует JS-файлы.
    gulp-plumber - отслеживание ошибок Gulp.
    gulp-notify - дополнение к plumber, выводит понятное уведомление об ошибке.
    gulp-imagemin@7.1.0 - сжимает картинки, обязательно 7.1 версия, 8-я забагованная.
    del - удаляет лишние файлы в dist дериктории, обязательно 6.1.1, 7-я забагованная.
    panini - генератор плоских файлов. Пока что безпонятия что это.
    gulp-concat - собирает файлы любых типов воедино.
    

Список команд:
    Работы с npm пакетами и Gulp происходит через терминал. Вызов терминала - Ctrl + ~.
    Команды для работы с терминалом:
        1. cd имя_папки - перемешение в указанную папку.
        2. cd .. - выход из папки.
        3. dir - список доступных файлов и папок в данном репозитории.
        4. mkdir -p название_папки - создание новой папки.

    npx команды:
        npx mkdirp имя_проекта - создание новой папки через npx.

    Стартовые, пошаговые команды для первичной настройки gulp:
        1. node --version (node -v) - проверка версии установленной платформы nodejs.
        2. npm -v - проверка версии npm.
        3. npx -v - проверка версии npx.
        4. npm install --global gulp-cli (npm i -g gulp-cli) - глобальная установка Gulp на компьютер.

    Настройка проекта под gulp:
        1. npm init - инициилизация npm, создает файл package.json в котором хранятся все настройки текущего проекта:
            Настройка npm init:
                1. package name: (имя_папки_проекта) - наименование проекта, в случае пропуска укажет имя в скобках.
                2. version: (1.0.0) - версия вашей сборки. Изначально всегда будет 1.0.0.
                3. description: - описание вашей сборки.
                4. entry point: (index.js) - корневой файл точки входа, изначально index.js.
                5. text command: ?
                6. git repository: - подключение удаленного репозитория Git.
                7. keywords: - ключевые слова.
                8. author: - имя автора сборки.
                9. license: (ISC) - Лицензия.
        1.2. npm init -y - альтернативный быстрый способ создания файла package.json.
        2. npm install --save-dev gulp (npm i --D gulp)- установка Gulp локально в проект.
    
    Команды Gulp: 
    1. gulp - запуск Gulp.
